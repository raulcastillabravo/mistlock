Esta guía te muestra cómo trabajar con **AWS S3** usando **Garage** como emulador local. Este lab demuestra un pipeline de datos usando **Boto3**, **PyArrow** y **Delta Lake** (delta-rs).

## Prerrequisitos

- [Docker](https://www.docker.com/get-started) instalado y en ejecución.
- [Dev Containers extension](vscode:extension/ms-vscode-remote.remote-containers) instalada (opcional).

## Cómo ejecutar

<Steps>
   1. **Configurar el entorno**:
      <Tabs>
         <TabItem label="Dev Container (recomendado)">
            Abre **VS Code** en la carpeta del proyecto y ejecuta este comando en la **Paleta de Comandos**:
            ```bash
            > Dev Containers: Reopen in Container
            ```
         </TabItem>
         <TabItem label="Manual">
            1. **Ejecuta el script** de configuración para instalar herramientas y dependencias.
               ```bash
               scripts/setup.sh
               ```
            2. **Iniciar la infraestructura**: Lanza los contenedores necesarios.
               ```bash
               docker compose up -d
               ```
         </TabItem>
      </Tabs>

   2. **Ejecutar el ejemplo**:
      ```bash
      python main.py
      ```
</Steps>

:::note[Claves S3 de Garage]
Las claves de conexión S3 de Garage se refrescan automáticamente al iniciar o reiniciar el entorno de desarrollo. Usa las generadas en tu `.env`, no las del repositorio.
:::

## Cómo depurar

1. **main.py**:
   - **Abre**: Abre `main.py`.
   - **Breakpoints**: Coloca breakpoints en el código.
   - **Ejecuta**: Pulsa `F5` para iniciar la depuración.

2. **Tests**:
   - **Abre**: Abre un fichero de test (p. ej., `tests/components/test_s3_boto.py`).
   - **Breakpoints**: Coloca breakpoints en el código del test.
   - **Ejecuta**: Usa la pestaña **Testing** de VS Code y haz clic en el icono **Debug Test** junto al test que quieras depurar.

## Cómo testear

- **Individualmente**: Ejecuta los tests individualmente desde la pestaña **Testing** de VS Code.
- **Todos los tests**: Ejecuta el script automatizado:
  ```bash
  scripts/run_tests.sh
  ```

## Validar resultados

<Tabs>
   <TabItem label="AWS CLI">
      - **Verifica los buckets**: Comprueba que se crearon los buckets `bronze` y `silver`.
        ```bash
        aws s3 ls --profile garage
        ```
      - **Verifica el contenido**: Comprueba los archivos en el bucket silver.
        ```bash
        aws s3 ls s3://silver --recursive --profile garage
        ```
   </TabItem>
   <TabItem label="Rclone">
      - **Lanza** la interfaz gráfica de Rclone:
        ```bash
        scripts/rclone-gui.sh
        ```
      - **Abre** en tu navegador:
        ```bash
        http://127.0.0.1:5572
        ```
   </TabItem>
</Tabs>

## Limpieza

```bash
docker compose down -v
```

## Solución de problemas

| Problema | Solución |
|----------|----------|
| API de Garage no lista | Asegúrate de que el contenedor `garage` está en ejecución y espera unos segundos a que la API se inicialice. Revisa los logs con `docker logs garage`. |
| Puerto 3900 en uso | Detén cualquier otro servicio que use el puerto 3900 o cambia el mapeo en `docker-compose.yml`. |
