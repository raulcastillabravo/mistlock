Esta guía te muestra cómo construir un escenario de nube híbrida usando **LocalStack** y una instancia externa de **PostgreSQL**. Este lab demuestra cómo una **AWS Lambda** recupera secretos de **Secrets Manager** e interactúa con una base de datos fuera del entorno AWS.

## Prerrequisitos

- [Docker](https://www.docker.com/get-started) instalado y en ejecución.
- [Dev Containers extension](vscode:extension/ms-vscode-remote.remote-containers) instalada (opcional).
- [Terraform](https://developer.hashicorp.com/terraform/install) (incluido en el Dev Container).
- [AWS CLI](https://aws.amazon.com/cli/) (incluido en el Dev Container).
- [DBeaver](https://dbeaver.io/) o cualquier cliente SQL (para verificar resultados).

## Cómo ejecutar

<Steps>

1. **Configurar el entorno**:
   <Tabs>
      <TabItem label="Dev Container (recomendado)">
         Abre **VS Code** en la carpeta del proyecto y ejecuta este comando en la **Paleta de Comandos**:
         ```bash
         > Dev Containers: Reopen in Container
         ```
      </TabItem>
      <TabItem label="Manual">
         ```bash
         docker compose up -d
         ```
      </TabItem>
   </Tabs>

2. A continuación, configura el **AWS CLI**:
   ```bash
   aws configure set aws_access_key_id test
   aws configure set aws_secret_access_key test
   aws configure set region us-east-1
   ```

3. **Desplegar recursos**:
   ```bash
   python package_lambda.py
   terraform init && terraform apply -auto-approve
   ```

4. **Ejecutar el ejemplo**:
   ```bash
   aws --endpoint-url=http://localhost:4566 lambda invoke \
       --function-name AddUserFunction \
       --payload '{}' \
       response.json
   ```

</Steps>

## Validar resultados

1. **Verifica con DBeaver**: Conecta a la instancia local de **PostgreSQL** y ejecuta la siguiente consulta:
   - **Host**: `localhost`.
   - **Port**: `5432`.
   - **Database**: `mydb`.
   - **User**: `myuser`.
   - **Password**: `mypassword`.

   ```sql
   SELECT * FROM users ORDER BY created_at DESC;
   ```

2. **Verifica con AWS CLI**:
   - **Lista las funciones Lambda**:
     ```bash
     aws --endpoint-url=http://localhost:4566 lambda list-functions
     ```
   - **Comprueba los secretos**:
     ```bash
     aws --endpoint-url=http://localhost:4566 secretsmanager list-secrets
     ```

## Limpieza

```bash
terraform destroy -auto-approve
docker compose down -v
rm lambda.zip response.json
```

## Solución de problemas

| Problema | Solución |
|----------|----------|
| Conexión rechazada | Asegúrate de que `host.docker.internal` es accesible desde el contenedor LocalStack. En versiones antiguas de Docker en Linux, añádelo a `extra_hosts` en `docker-compose.yml`. |
| Puerto ya en uso | Modifica los mapeos de puertos para `4566` o `5432` en `docker-compose.yml`. |
