Esta guía te muestra cómo implementar un mutex distribuido usando **Redis**, **Docker Compose** y **Python**. Este lab demuestra cómo coordinar el acceso concurrente a recursos compartidos entre múltiples hilos.

## Prerrequisitos

- [Docker](https://www.docker.com/get-started) instalado y en ejecución.
- [Dev Containers extension](vscode:extension/ms-vscode-remote.remote-containers) instalada (opcional).

## Cómo ejecutar

<Steps>

1. **Configurar el entorno**:
   <Tabs>
      <TabItem label="Dev Container (recomendado)">
         Abre **VS Code** en la carpeta del proyecto y ejecuta este comando en la **Paleta de Comandos**:
         ```bash
         > Dev Containers: Reopen in Container
         ```
      </TabItem>
      <TabItem label="Manual">
         ```bash
         pip install uv && uv sync
         ```
      </TabItem>
   </Tabs>

2. **Iniciar la infraestructura**: Lanza el contenedor de Redis:
   ```bash
   docker compose up -d
   ```

3. **Ejecutar el ejemplo**:
   <Tabs>
      <TabItem label="Lock/Unlock">
         ```bash
         python main.py
         ```
      </TabItem>
      <TabItem label="Context Manager">
         ```bash
         python main_acquire.py
         ```
      </TabItem>
   </Tabs>

</Steps>

## Validar resultados

**Verifica** que cada hilo escribió su línea en orden comprobando `shared_file.txt`:
```bash
cat shared_file.txt
```

## Limpieza

```bash
docker compose down -v
rm shared_file.txt
```
