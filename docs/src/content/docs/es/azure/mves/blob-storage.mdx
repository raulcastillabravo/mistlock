Esta guía te muestra cómo trabajar con **Azure Blob Storage** localmente usando **Azurite**, **Docker Compose** y **Python**. Este lab demuestra cómo crear contenedores y subir y descargar blobs.

## Prerrequisitos

- [Docker](https://www.docker.com/get-started) instalado y en ejecución.
- [Dev Containers extension](vscode:extension/ms-vscode-remote.remote-containers) instalada (opcional).
- [Azure Storage extension](vscode:extension/ms-azuretools.vscode-azurestorage) (opcional, incluida en el Dev Container).

## Cómo ejecutar

<Steps>

1. **Configurar el entorno**:
   <Tabs>
      <TabItem label="Dev Container (recomendado)">
         Abre **VS Code** en la carpeta del proyecto y ejecuta este comando en la **Paleta de Comandos**:
         ```bash
         > Dev Containers: Reopen in Container
         ```
      </TabItem>
      <TabItem label="Manual">
         1. **Ejecuta el script** de setup para instalar herramientas y dependencias.
            ```bash
            scripts/setup.sh
            ```
         2. **Inicia la infraestructura**: Lanza el contenedor de Azurite.
            ```bash
            docker compose up -d
            ```
      </TabItem>
   </Tabs>

2. **Ejecutar el ejemplo**:
   ```bash
   python main.py
   ```

</Steps>

## Cómo depurar

1. **Script de Python (main.py)**:
   - **Abre**: Abre `main.py` en VS Code.
   - **Ejecuta**: Usa la configuración **Python: main.py** en la pestaña **Run and Debug**.

## Cómo testear

- **All tests**: Ejecuta el script automatizado: `scripts/run_tests.sh`
- **Individually**: Usa la pestaña **Testing** de VS Code para ejecutar o depurar tests concretos.

## Validar resultados

**Verifica con la [extensión Azure Storage](https://marketplace.visualstudio.com/items?itemName=ms-azuretools.vscode-azurestorage)**:
- **Abre**: Abre la vista **Azure** en VS Code (icono de Azure en la barra lateral).
- **Conecta**: En **Storage Accounts**, haz clic en **Attach Storage Account** > **Attach to a local emulator**.
- **Navega**: Usa la configuración predeterminada de Azurite para explorar visualmente tus contenedores y blobs.


## Limpieza

```bash
docker compose down -v
```

## Solución de problemas

| Problema | Solución |
|----------|----------|
| Puerto 10000 en uso | Actualiza el mapeo de puertos en `docker-compose.yml` y `BlobEndpoint` en `.env`. |
| Conexión rechazada | Asegúrate de que Azurite está en ejecución: `docker ps` |
| Módulo no encontrado | Reinstala las dependencias: `scripts/setup.sh` |
