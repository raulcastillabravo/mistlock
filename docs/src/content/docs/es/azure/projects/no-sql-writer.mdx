Esta guía te muestra cómo trabajar con **Azure Cosmos DB** localmente usando **Docker** y **Python**. Este lab demuestra cómo conectar una aplicación Python al emulador de Azure Cosmos DB.

## Prerrequisitos

- [Docker](https://www.docker.com/get-started) instalado y en ejecución.
- [Dev Containers extension](vscode:extension/ms-vscode-remote.remote-containers) instalada (opcional).

## Cómo ejecutar

<Steps>

1. **Configurar el entorno**:
   <Tabs>
      <TabItem label="Dev Container (recomendado)">
         Abre **VS Code** en la carpeta del proyecto y ejecuta este comando en la **Paleta de Comandos**:
         ```bash
         > Dev Containers: Reopen in Container
         ```
      </TabItem>
      <TabItem label="Manual">
         1. **Instala las dependencias**:
            ```bash
            pip3 install uv && uv sync
            ```
         2. **Iniciar la infraestructura**: Lanza el emulador de Cosmos DB:
            ```bash
            docker compose up -d cosmos
            ```
      </TabItem>
   </Tabs>

2. **Ejecutar el ejemplo**:
   ```bash
   python main.py
   ```

</Steps>

:::note[Tiempo de arranque del emulador]
Espera **15-30 segundos** después de iniciar el emulador antes de ejecutar `python main.py` para que todos los servicios internos se inicialicen.
:::

## Validar resultados

<Tabs>
   <TabItem label="Web Explorer">
      **Abre** el explorador de Cosmos DB en tu navegador:
      ```bash
      http://localhost:1234/
      ```
   </TabItem>
   <TabItem label="Azure Cosmos DB Extension">
      **Instala** la [extensión Azure Cosmos DB](https://marketplace.visualstudio.com/items?itemName=ms-azuretools.vscode-cosmosdb) desde el marketplace de VS Code y conecta usando `COSMOS_ENDPOINT` y `COSMOS_KEY` definidos en tu `.env`.
   </TabItem>
</Tabs>

## Limpieza

```bash
docker compose down -v
```

## Solución de problemas

| Problema | Solución |
|----------|----------|
| Conexión rechazada | Asegúrate de que el emulador está en ejecución: `docker ps` |
| `PostgreSQL=FAIL` en los logs | Reinicia el estado del emulador: `docker compose down -v` |
| Período de evaluación expirado | Elimina la imagen antigua (`docker rmi mcr.microsoft.com/cosmosdb/linux/azure-cosmos-emulator`), descarga `latest` y reinicia. |
