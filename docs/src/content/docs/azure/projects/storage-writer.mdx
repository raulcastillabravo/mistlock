This guide shows you how to work with **Azure Functions** locally using **Azurite**, **Docker Compose**, and **Python**. This lab demonstrates how to create an HTTP-triggered function that uploads files to Azurite blob storage.

## Prerequisites

- [Docker](https://www.docker.com/get-started) installed and running.
- [Dev Containers extension](vscode:extension/ms-vscode-remote.remote-containers) installed (optional).

## How to execute

<Steps>

1. **Setup Environment**:
   <Tabs>
      <TabItem label="Dev Container (recommended)">
         Open **VS Code** in the project folder and execute this command in the **Command Palette**:
         ```bash
         > Dev Containers: Reopen in Container
         ```
      </TabItem>
      <TabItem label="Manually">
         1. **Install Azure Functions Core Tools**: Follow the [official installation guide](https://learn.microsoft.com/en-us/azure/azure-functions/functions-run-local).
         2. **Install dependencies**:
            ```bash
            pip3 install uv && uv sync
            ```
      </TabItem>
   </Tabs>

2. **Start Infrastructure**: Launch the Azurite container:
   ```bash
   docker compose up -d
   ```

3. **Start Function Host**:
   ```bash
   func start --prefix ./upload-function/
   ```

4. **Run the Example**:
   ```bash
   python main.py
   ```

</Steps>

## Validate results

**Check using [Azure Storage extension](https://marketplace.visualstudio.com/items?itemName=ms-azuretools.vscode-azurestorage)**:
- **Open**: Open the **Azure** view in VS Code (Azure icon in the sidebar).
- **Connect**: Under **Storage Accounts**, click **Attach Storage Account** > **Attach to a local emulator**.
- **Browse**: Use the default Azurite connection settings to browse your containers and blobs visually.

## Clean Up

```bash
docker compose down -v
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Connection Refused | Ensure Azurite is running: `docker ps` |
| Module Not Found | Reinstall function dependencies: `cd upload-function && pip install -r requirements.txt` |
